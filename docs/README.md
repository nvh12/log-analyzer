| Use Case                            | What is Detected                                                   | Method                                                                                           | Possible Log Dataset                                              | Possible Reaction                                             |
| ----------------------------------- | ------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------ | ----------------------------------------------------------------- | ------------------------------------------------------------- |
| **Exception spike detection**       | Sudden increase in application exceptions                          | Rule engine (threshold + sliding window)                                                         | Application logs with log level + exception name                  | Send alert, create incident, attach recent logs for debugging |
| **High HTTP error rate**            | Large percentage of 5xx responses                                  | Rule engine (error ratio in time window)                                                         | Web server / API gateway logs (status codes, endpoint, timestamp) | Alert ops team, rollback deployment, enable circuit breaker   |
| **Slow API endpoints**              | Average or p95 response time exceeds threshold                     | Rule engine or statistical anomaly detection (z-score / baseline deviation)                      | API access logs with response time field                          | Alert, recommend scaling, flag slow endpoint                  |
| **Slow database queries**           | Queries exceeding latency threshold                                | Rule engine (slow query threshold)                                                               | Database slow query logs or application DB logs                   | Alert DB team, store slow queries for optimization            |
| **Traffic spike / possible DDoS**   | Unusual increase in total request rate                             | ML anomaly detection (Isolation Forest, ARIMA time-series anomaly detection) or simple threshold | Web access logs (timestamp, IP, endpoint)                         | Rate limiting, activate WAF rules, alert security             |
| **IP request flooding**             | Single IP sending excessive requests                               | Rule engine (per-IP sliding window counter)                                                      | Web server logs (IP address, request timestamp)                   | Temporarily block IP, apply rate limiting                     |
| **Brute force login attack**        | Many failed login attempts for one account                         | Rule engine (failed login count per user/IP)                                                     | Authentication logs (username, result, IP)                        | Lock account temporarily, require CAPTCHA, notify user        |
| **Credential stuffing attack**      | Many login attempts across many accounts from same IP              | Rule engine or clustering anomaly detection                                                      | Authentication logs (username, IP, login result)                  | Block IP, enforce MFA, alert security                         |
| **Dependency failure detection**    | Repeated connection errors to services (DB, cache, message broker) | Rule engine (error pattern detection)                                                            | Application logs with dependency error messages                   | Trigger fallback mechanism, alert operations                  |
| **Retry storm / cascading failure** | Services repeatedly retry failing operations                       | Rule engine (retry log pattern detection)                                                        | Application logs with retry messages and timestamps               | Throttle retries, activate circuit breaker                    |
| **Unusual access pattern**          | Rare or abnormal API call sequences                                | ML sequence anomaly detection (LSTM, HMM, or clustering)                                         | API access logs (user/session ID, endpoint sequence)              | Alert security, flag suspicious session                       |
| **Log anomaly detection**           | Previously unseen log message patterns                             | ML anomaly detection (log clustering, DeepLog / autoencoder)                                     | Raw application logs (structured or parsed)                       | Alert engineers for investigation                             |
